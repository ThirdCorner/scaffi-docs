Project Setup

Let's make a project

Scaffi has a lot of baked in structure in the framework, so it leverages Yeoman to handle any file generation so that we don’t need to copy/paste whenever we’re adding something to a project. It even comes with a generator to scaffold out a new project so let’s do that. In your CMD window, navigate to your projects directory, wherever you want this tutorial project to go. Once you’re there, run the following:

yo scaffi

This will start Yeoman and run you through project creation. For the sake of this tutorial, do the following so we can have a basis to work with:

  1. Hit enter at the first prompt
  2. Hit enter at “New Project/Add to Project”
  3. Hit enter for “Both”
  4. Project Name: rocking-form
  5. Add your name
  6. Type “api” (This is setting up the base route, as in where to direct all api server requests when not in prototype mode made by the UI)
  7. Type 5010 (This is the UI location in the browser when you’re developing.)
  8. Type 3000 (This is the Localhost Server location the UI calls to make requests)
  9. Hit enter for defaults (If we had a database/api that we have to hook into, we might customize here to tell Scaffi how to parse data structures coming in from the server)
  10. This will now run npm/jspm install on the UI and the Server. Go get coffee, this will take a bit.
  11. Once that’s all installed, it’s going to have you pick a theme. Again, hit enter and then select Angular Material (Scaffi can do both material and bootstrap, so you’d pick whichever you need for your project. But, material is definitely worth it if you can use it)
  12. Select "basic" when it asks you what type of page you want to make.

After all that installation and setup, you should be back at the CMD prompt. Type the following:

cmd rocking-form
yo scaffi:mode

This command is for switching between various environment modes. Everything but prototype mode expects a backend service to talk to. For now, hit enter, select Prototype, and hit enter again.

To start the UI, in your CMD window, run the following:

yo scaffi:start ui

If all was good with your setup, this will start some gulp processes. Give it a minute and it’ll launch a browser tab going to http://localhost:5010. It should look like this, unless we’ve added a theme to the generators:

Whenever you run the app in the browser, make sure to open the console window in Chrome. We have browserify watching for changes, so as long as you have the console open, you can make changes and the page will auto-refresh. If however, you make changes and then start up yo scaffi:start ui, you want to make sure your console is open and then refresh the page. This will get a fresh copy of your changes. So remember, if you end up wondering “Where the hecks the changes I just made”, make sure your console is open, caching is disabled in network tab in chrome, and you’ve manually refreshed for the first time.

Congrats. You now have a fully scaffolded template project to edit to your hearts content. Let’s dig into this whole prototype mode thingymajigger.

In your favorite IDE, open up the project source that got generated. We only care about the UI code right now, so take a look in rocking-form/src/ui. Should look like a standard project setup, except with maybe a couple more json files than you’re used to. The angular app source is in the app folder, in which you will see your typical angular module concepts. Once you’ve taken a look around, let’s highlight some distinctions that Scaffi makes:

  • These are html tag type components, like a date picker or whatnot that are used throughout the app in an html template.
  • These are html tag attribute components. These don’t have a template associated with them.
  • These are general singleton classes in your app that you can call. For instance, an ErrorHandling class would be a factory, since you want to be able to use it anywhere in your app's controllers.
  • Where all your pages and their controllers live. The structure of the routes folder will match your browser url. If you had a url localhost:5010/products/edit, your file path in routes would look like routes/products/edit/index. The homepage is in the routes/index folder.
  • These are your http calls to your server and any other external systems you might need to hook up to. Services have classic REST commands as well as a few others.
  • Where your header, footer, layout, and styles are all grouped. Sass styles are all in theme/styles folder so put all your custom layout design in that thar place.
