Setup an Existing App

If you ever have to run an existing app, there's a few steps you’ll need to follow to get things running:

  1. Make sure you have yo and generator-scaffi installed globally. Refer to the Installation Guide on the actual commands.
  2. Check out the project from svn/git; You'll probably need to switch to the develop branch, but check with your project lead about that.
  3. Go into src/server/config and create a scaffi-server.localhost.private.json
    1. You’ll need to either reference another config to see what properties the project is using, or ask an existing member to share theirs.
    2. Environment in the config needs to be localhost
  4. Go into src/ui/config and create a scaffi-ui.localhost.private.json. Again reference an existing config or ask for one.
    1. Environment needs to be localhost
    2. You don’t need domain or mobileDomain for localhost unless your apis are located externally.
  5. Run yo scaffi:mode localhost to switch localhost configs into use.
  6. If you need to seed a database, go into src/server/scaffi-server.private.json
    1. In the components.sequelize object, you want to add the following property:
      1. sync: true
  7. Run yo scaffi:start web (or whatever platform you want to work on); This will do any necessary npm installs for the server and ui. This will also eventually seed your local database if there are no sequelize errors.
  8. Once your app opens in a tab, if you synced a database, you’ll need to go back into src/server/scaffi-server.private.json and remove the sync property. When you save, this will restart the server.

You should be good after that.
