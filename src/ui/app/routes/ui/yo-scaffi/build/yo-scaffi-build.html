yo scaffi:build

Call this when you want to build an app into a certain platform and environment.

Usage

yo scaffi:build [platform] [environment]

Platforms: web, ios, android

Environment: prototype, localhost, development, ci, qa, production

In-depth Description

As a broad overview, when this command is run, it bundles up any necessary app resources (js, html, scss), and runs browserify on the app.js to generate the app script bundle.

Building Web

yo scaffi:build web [environment] --version=###

When a web build happens, the resulting server + ui bundle will end up in /builds/web/server. If a web.config does not exist, the generator will add a basic one in case the server is being run by IIS. IISNode will need to be installed separately, however, the server should then load inside IIS and is hooked up to receive the port passed by IIS in that instance.

Building Android/iOS

yo scaffi:build ios [environment] --version=### --name=AppName --namespace=com.*.*

yo scaffi:build android [environment] --version=### --name=AppName --namespace=com.*.*

--name: What to name the app once it's installed on a device

--namespace: This is most prevalent with ios; This is the corresponding app_id associated with a provisioning profile.

When building a mobile version, the generators will copy over the /src/ui/mobile folder into the /src/ui/build/[platform]/public folder, bundle all resources, run ionic build, and then transfer the resulting ipa / apk into the /builds/[platform]/public folder. If you're using an automated build process that will sign the app, you can run any additional commands on the /src/ui/build/[platform]/ folder as if it were an ionic project.
