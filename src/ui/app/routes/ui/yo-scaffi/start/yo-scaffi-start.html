yo scaffi:start ui

Call this when you want to start your app in a local environment. This will start your front-end app and server at the same time so that your app can function correctly.

Usage

yo scaffi:start [platform]

Platforms: web, ios, android

In-depth Description

Use this command when you want to test code or run the app locally. Based on the yo scaffi:mode command you call before, the app will run with whatever config is set. When the start command runs, it will check if your current node_modules are up-to-date with the app's package.json for both UI and Server (in case someone installed or uninstalled packages), it will run the build process to package everything, and then it will run the Server and BrowserSync instances so your app can function. LiveReload is automatically enabled, and will rebuild js and html before reloading the browser automatically. For sass changes, the styles are auto-injected without having to reload.

For mobile platforms, there's not really any distinction between ios and android, unless you have mobile platform-specific packages installed for your app. The start command will open a browser window as if you were running ionic serve directly. So if you're just looking to test mobile as a general environment, pick your favorite platform between the two and run one. You'll be essentially testing both simultaneously (barring any os specific bugs).
