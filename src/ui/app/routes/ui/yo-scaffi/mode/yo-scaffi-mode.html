yo scaffi:mode

Call this when you want to set the environment for both UI and Server.

Environments

  • Used by the UI to intercept all requests to the server and redirect to in-memory storage. This lets you store all data in memory and prototype out your site without having to worry about a backend.
  • Used for defining when you’re developing locally. You can use this mode to return mock data for an API if you don’t want to deal with the connection hassel.
  • The mode to set when you’re running your environment under continuous integration development.
  • A good mode to use when you need to test implementation on the actual accessible server.
  • A good mode to use when you’re having other people run through features on a like-production esque server.
  • The mode to set when you’re running your code in a live environment.
