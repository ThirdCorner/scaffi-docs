Routes(Modules)

A module in Angular terms determines a grouping of pages and how to get to them. This is where we integrate UI-Router to build our app page structures.

This is an example of how we do modules in Angular 1.x

file: app/components/billing/billingModule.js

In this snippet, we’re registering a namespace with Angular that, when called, will set up the billing routes for the app. This tells Angular what url defines each page and what controller to call.

In the new structure, we split this concept into routes and pages.

A route in the new structure groups a collection of pages, just as a namespaced module grouped a collection of states.

For example, in a url such as http://github.com/git/edit, git, is a route, where as editTime is a page.

All routes, when generated with yo scaffi:route get a page called index so that the route can stay abstract in ui-router terms, and all pages under a route get their own folder in the filesystem tree structure.

Let's take a look at the structure of a route if we rewrote the billingModule module above:

file: app/routes/billing-report/billing-report.route.js

The general gist of this snippet is that we have params which define the details of a route decorator which attaches to a Javascript class. This class is a controller in the Angular sense, so you can put logic in here that runs whenever someone first comes to this route.

If the PARAMS const look familiar, they’re the exact same ones you would provide a state in ui-router. Some params are filled in for you via the decorator, like the controller param, but if you want more information on what you can set in the state, you can go to: https://github.com/angular-ui/ui-router/wiki

The main purpose of this route, besides grouping pages into logical categories, is to redirect to the index page whenever this route is called specifically. It’s a nice user enhancement so that if they happen to go to http://webapp.com/billing-report, ui-router will redirect them to http://webapp.com/billing-report/. This becomes important when you add breadcrumbs because breadcrumbs will go to the former url, not the latter.
