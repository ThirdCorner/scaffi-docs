res

There's a few added functions to the res (response) object we get in express (http://expressjs.com/en/4x/api.html#res).

res.sendSuccess(data)

Use when you want to return a json success to the api. If you send a string, this function will package it into a json object.

Name Type Description
data Scalar
|| Object
|| Array		 The response data you want to send.

res.sendForbidden(errorMsg, stack)

Use when you want to send a 403 HTTP response to the UI with troubleshoot data.

Name Type Description
errorMsg Scalar
|| Object
|| Array		 Whatever error message data you want to send back to the UI.
stack Scalar
|| Object
|| Array		 optional For use if you have a stacktrace you want to send back.

res.sendUnauthorized(errorMsg, stack)

Use when you want to send a 401 HTTP response to the UI with troubleshoot data.

Name Type Description
errorMsg Scalar
|| Object
|| Array		 Whatever error message data you want to send back to the UI.
stack Scalar
|| Object
|| Array		 optional For use if you have a stacktrace you want to send back.

res.sendError(errorMsg, stack)

Use when you want to send a 500 HTTP response to the UI with troubleshoot data.

Name Type Description
errorMsg Scalar
|| Object
|| Array
|| Error		 Whatever error message data you want to send back to the UI. You can also pass an Error class object.
stack Scalar
|| Object
|| Array		 optional For use if you have a stacktrace you want to send back. If you send an Error object in errorMsg, the stacktrace will get pulled out automatically.

res.sendNotFound(errorMsg, stack)

Use when you want to send a 404 HTTP response to the UI with troubleshoot data.

Name Type Description
errorMsg Scalar
|| Object
|| Array		 Whatever error message data you want to send back to the UI.
stack Scalar
|| Object
|| Array		 optional For use if you have a stacktrace you want to send back.
